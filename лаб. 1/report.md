## Решение на Этап 3. Мини-проект: Веб-приложение для учета настольных игр (Flask)

### 1. Выбор методологии и обоснование

Для реализации проекта была выбрана методология **Agile (Scrum)**.

**Обоснование:**
Разработка веб-приложения для учета настольных игр предполагает постепенное расширение функциональности: от базового хранения данных до более сложных возможностей (фильтрация, категории, рейтинг и т.д.). Методология Agile позволяет:

* быстро реализовать **MVP** (минимально жизнеспособный продукт) — добавление и просмотр игр;
* итеративно расширять функционал (редактирование, удаление, поиск);
* тестировать каждую часть независимо;
* гибко реагировать на изменения требований.

---

### 2. Этапы разработки (в соответствии с Agile)

Проект был разбит на три спринта:

1. **Спринт 1 (Backend Core):**

   * Проектирование структуры базы данных для хранения настольных игр;
   * Настройка административной панели Django.

2. **Спринт 2 (User Interface):**

   * Создание страниц для отображения списка игр;
   * Реализация страницы с подробной информацией об игре.

3. **Спринт 3 (Interactivity / CRUD):**

   * Добавление функционала:

     * добавление новой игры;
     * редактирование существующей;
     * удаление игры.

---

### 3. Выбор и обоснование методов для каждого этапа

| Этап                       | Метод / Инструмент                     | Обоснование                                                                                                                |
| -------------------------- | -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **Проектирование БД**      | **ORM (Object-Relational Mapping)**    | Позволяет описывать настольные игры через Python-классы без написания SQL, ускоряя разработку и снижая вероятность ошибок. |
| **Архитектура приложения** | **Паттерн MVT (Model-View-Template)**  | Обеспечивает разделение логики, данных и представления, что упрощает поддержку и масштабирование проекта.                  |
| **Интерфейс**              | **Django Templates**                   | Позволяет динамически выводить список игр и их детали на HTML-страницах.                                                   |
| **Работа с формами**       | **Django Forms**                       | Упрощает обработку пользовательского ввода и валидацию данных при добавлении/редактировании игр.                           |
| **Тестирование**           | **Ручное функциональное тестирование** | Проверка корректности CRUD-операций (создание, просмотр, редактирование, удаление игр).                                    |

---

### 4. Реализация проекта

#### Краткая документация (структура приложения)

**Model (Модель):**
Класс `BoardGame`, описывающий настольную игру:

* `title` — название игры
* `description` — описание
* `genre` — жанр (например: стратегия, семейная)
* `players` — количество игроков
* `created_at` — дата добавления

**View (Представление):**

* отображение списка игр;
* просмотр отдельной игры;
* создание, редактирование и удаление игр.

**Template (Шаблоны):**

* HTML-страницы для:

  * списка игр;
  * страницы одной игры;
  * формы добавления/редактирования.

---

### Заключение

В ходе выполнения мини-проекта было разработано веб-приложение для учета настольных игр, позволяющее:

* сохранять имеющиеся игры;
* добавлять новые;
* просматривать список игр;
* редактировать информацию;
* удалять игры.

Применение методологии Agile обеспечило поэтапную разработку и возможность быстрого расширения функциональности. Использование методов ORM и архитектуры MVT позволило упростить работу с базой данных и обеспечить структурированность кода.

Проект может быть расширен в будущем за счет добавления:

* системы пользователей;
* рейтингов и отзывов;
* фильтрации и поиска игр.
